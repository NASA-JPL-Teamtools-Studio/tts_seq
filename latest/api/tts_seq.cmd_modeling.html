class tts_seq.cmd_modeling.commands.Command(module, seq_step, sequence_engine_id=None)

Bases: object

Base class for modeling the execution of a spacecraft command.

A Command represents the simulated lifecycle of a hardware or software action. It breaks down a single sequence command into one or more discrete CommandStep objects that are processed sequentially by the parent module’s simulation loop.

Parameters:

  • module (Module) – The simulation module instance handling this command.

  • seq_step (SeqStep) – The raw sequence step data from the sequence engine.

  • sequence_engine_id (int, optional) – Index of the calling sequence engine, if any.

class tts_seq.cmd_modeling.commands.CommandStep(module, *args, **kwargs)

Bases: object

Abstract base class for an atomic unit of simulation logic within a Command.

CommandSteps are executed one at a time. The next step in a Command will not start until the current step’s complete attribute is set to True.

Parameters:

module (Module) – The parent module instance.

simulate()

Logic to be executed every simulation second. Subclasses must override this and set self.complete = True when the step is finished.

class tts_seq.cmd_modeling.commands.LinearToGoal(module, goal_label, actual_label, rate_per_s)

Bases: CommandStep

Models a value transitioning linearly over time until it reaches a goal.

Commonly used to simplistically model temperature changes or motor movements.

Parameters:

  • goal_label (str) – Attribute name on the module containing the target value.

  • actual_label (str) – Key in simulation modeled_values to be updated.

  • rate_per_s (float) – The amount to change the actual value per simulation second.

simulate()

simulation logic to step towards a goal. If the value is at the goal, step compeltes. If it is less than one time step away, it jumps to the goal and completes. Otherwise, it takes one time step towards the goal.
