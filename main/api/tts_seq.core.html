Reader and container for standard .seq (dot-seq) sequence files common on Earth orbiting missions.

This class handles the parsing of legacy ASCII sequence files, which use positional arguments and special character-based metadata (semicolons). It leverages a Command Dictionary to map positional values found in the text file back to named arguments with specific bit-lengths and types.

resolve_steps(entry_point, ancestor_seqs=None, sequence_begin_time=None, expand_subsequences=True)

Recursively resolves sequence calls into a flat list of steps with absolute timestamps.

This method processes the ‘entry_point’ sequence and, if any commands in that sequence are “call” commands, it recursively resolves those sub-sequences as well (unless expand_subsequences is False).

This is of limited use for projects that use any sort of conditional sequencing. In that case, use tts_seq.core.simulation. But for projects that use only absolute and relative time commanding, this can be used.

That being said, it’s likely that if you want this, you also want other simulation capabilities, in which case you’d want tts_seq.core.simulation.

Parameters:

  • entry_point (str) – The ID of the sequence to start resolution from.

  • ancestor_seqs (list[str], optional) – List of sequence IDs that led to the current sequence (for tracking genealogy).

  • sequence_begin_time (datetime, optional) – The start time used for resolving relative tags.

  • expand_subsequences (bool) – If True, recursively expand sub-sequences. If False, only resolve times for the main sequence.

Raises:

  • Exception – If a relative command is found without a start time reference.

  • NotImplementedError – If an unsupported time type is encountered.
